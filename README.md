## Overview

CareerPilot turns any learning goal into a concrete, actionable plan — whether the user wants to become a software engineer, prepare for a university exam, pick up a new instrument, or master a practical life skill. It combines **generative AI** for roadmap creation with a **real-time voice AI agent** for interview practice, wrapped in a fast, modern Next.js App Router architecture with server and client rendering working together.

The platform isn't limited to tech careers: the same roadmap, quiz, and progress-tracking engine works for **academic subjects, professional certifications, and practical/hands-on skills**, making it a general-purpose AI learning companion rather than a single-niche tool.

Built as a full-stack, unified Next.js application — no separate backend service to deploy or maintain — while still integrating a real database layer, authentication provider, and multiple third-party AI services.

## ✨ Features

- **AI-Generated Learning Roadmaps** — Structured, phase-based learning plans (Beginner → Advanced) tailored to any goal — a tech career, an academic subject, a certification, or a practical skill — based on experience level and time commitment, powered by Google Gemini.
- **AI Voice Mock Interviews** — Real-time, spoken interview and oral-exam practice via Vapi's voice AI agents, with natural conversational flow instead of static Q&A forms.
- **AI-Generated Quizzes** — On-demand quizzes generated by Gemini based on the user's current roadmap phase, used to validate understanding before moving to the next milestone — technical, academic, or practical.
- **Interview Question Bank** — AI-generated bank of goal-specific interview and assessment questions, giving users a written reference to study alongside voice-based practice.
- **Progress Tracking** — Persistent tracking of roadmap completion, quiz performance, and practice history, so users can see measurable progress over time, regardless of skill domain.
- **Roadmap Scheduling** — Interactive calendar view (Schedule-X) so users can plan milestones and study sessions against real dates.
- **Secure Authentication** — Firebase Authentication on the client, verified server-side with Firebase Admin SDK for protected routes and API handlers.
- **Persistent User Data** — MongoDB for storing roadmaps, progress, and interview history.
- **Smooth, Modern UI/UX** — GSAP and Framer Motion-driven animations, DaisyUI components on top of Tailwind CSS v4, fully responsive across devices.
- **Efficient Data Fetching** — TanStack Query for caching, request deduplication, and background revalidation.
- **Type-Safe Validation** — Zod schemas for validating AI output and API payloads end-to-end.

## 🧠 How It Works

1. **User Input** — The user specifies a learning goal (e.g., "MERN Stack Developer," "Organic Chemistry for finals," or "conversational Spanish"), current experience level, and available time commitment.
2. **AI Prompting** — The app constructs a structured prompt and sends it to Gemini (via `@google/genai` / `@ai-sdk/google`).
3. **Structured Generation** — Gemini returns a validated (Zod-parsed) roadmap containing:
   - Learning phases
   - Core topics/technologies per phase
   - Suggested projects or practice exercises
   - Curated resources
   - Milestones
4. **Rendering** — The roadmap is rendered into an interactive UI, and optionally scheduled on the user's calendar.
5. **Quiz Generation** — At each phase, the user can request an AI-generated quiz to test their understanding before advancing.
6. **Interview/Assessment Prep** — The user can browse an AI-generated bank of goal-specific questions, or launch a live voice session with the Vapi AI agent to simulate a real spoken interview or oral assessment.
7. **Progress Tracking** — Roadmap completion, quiz scores, and practice activity are persisted and surfaced back to the user so they can track growth over time.

## 📦 Getting Started

### Prerequisites

- Node.js 18.18+ (Node 20 LTS recommended)
- A MongoDB instance (local or Atlas)
- A Firebase project (Auth + Admin SDK credentials)
- A Google Gemini API key
- A Vapi AI account/API key (for voice interviews)

### 1. Clone the repository

```bash
git clone https://github.com/alvy00/eg-careerpilot-asyncawait.git
cd eg-careerpilot-asyncawait
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure environment variables

Create a `.env.local` file in the project root:

```bash
# Database
MONGODB_URI=your_mongodb_connection_string

# Firebase (client)
NEXT_PUBLIC_FIREBASE_API_KEY=your_firebase_api_key
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your_project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your_project_id

# Firebase (server / Admin SDK)
FIREBASE_ADMIN_CLIENT_EMAIL=your_service_account_email
FIREBASE_ADMIN_PRIVATE_KEY=your_service_account_private_key

# AI — Gemini
GOOGLE_GENERATIVE_AI_API_KEY=your_gemini_api_key

# AI — Voice Interviews
NEXT_PUBLIC_VAPI_WEB_TOKEN=your_vapi_public_key
VAPI_API_KEY=your_vapi_private_key
```

### 4. Run the development server

```bash
npm run dev
```

The app will be available at [http://localhost:3000](http://localhost:3000).

Direct pushes to `main` are disabled. All changes go through a pull request:

1. **Sync with the latest development branch:**
   ```bash
   git checkout development
   git pull origin development
   ```

2. **Create a feature branch:**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Commit your changes** using clear, descriptive messages:
   ```bash
   git add .
   git commit -m "feat: add roadmap milestone editing"
   ```

4. **Push your branch:**
   ```bash
   git push origin feature/your-feature-name
   ```

5. **Open a Pull Request** on GitHub:
   - **Base branch:** `development`
   - **Compare branch:** your feature branch
   - Include a clear description of what changed and why.

6. Once reviewed and approved, the PR can be merged.

**Guidelines:** Keep commits and PRs small and focused. Ask questions early rather than getting stuck. Prefer conventional commit prefixes (`feat:`, `fix:`, `chore:`, `docs:`) where practical.

## 🔧 Troubleshooting

| Problem | Solution |
|---|---|
| `npm install` fails | Confirm Node.js 18.18+ and npm are installed and up to date |
| Port already in use | Stop the conflicting process or run `next dev -p <port>` |
| Firebase auth errors | Double-check `.env.local` values match your Firebase project config exactly |
| Gemini API errors | Verify your API key and check Google AI Studio quota/usage limits |
| Vapi voice session fails to start | Confirm microphone permissions and that the public Vapi token is exposed via `NEXT_PUBLIC_` |
| Can't push to `main` | Expected — open a Pull Request against `development` instead |
